Program​ Repair​ Golf

The application presents users with ​ repair challenges ​ , consisting of a program's
source code, and a repair objective. The goal for the user will be to change the source code to
achieve the objective, with the minimum possible number of changes to the original code. The
shorter the repair, the better. The user will collect points from every challenge he/she is able to
repair,​ ​ with​ ​ more​ ​ points​ ​ being​ ​ assigned​ ​ as​ ​ repairs​ ​ are​ ​ shorter.

The application allows users
to register, and maintains a record for each registered user based on the points he/she earns by
resolving repair challenges. A repair challenge consists of a program's source code, and a
repair objective, that the user must achieve by modifying the source code. The user is awarded
points based on the complexity of the challenge, and the length of the repair, with a "the shorter
the better" policy. Repair goals can vary, from simple "make this program compile" to more
complex ones, such as "make this program pass all tests" and "make this program correctly sort
an​ ​ array".

The challenges can only be created by registered users.

A​ challenge​ ​ involves:
● a name (Denomination name for the challenge)
● a language (Programming language in which is written the challenge)
● a difficulty (Determines difficulty of challenge from very easy to very hard)
● a type (compilation challenge, test passing challenge, verification challenge)
● a description (Describes what it’s the challenge about)
● a​ ​ program's​ ​ source​ ​ code

Let the game begin!
